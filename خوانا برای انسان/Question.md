# نمایش «خوانا برای انسان» حجم فایل

**محدودیت زمان:** ۱ ثانیه  
**محدودیت حافظه:** ۲۵۶ مگابایت  

سِلیب یک فایل به اندازه‌ی **m بایت** روی هارد دارد و می‌خواهد حجم تقریبی آن را به فرم «خوانا برای انسان» نمایش دهد.

نمایش «خوانا برای انسان» به صورت **`nU`** است و از دو بخش تشکیل شده است:

### شرایط بخش‌ها
- **عدد (n)** باید یک عدد صحیح در بازه‌ی `[1, 1023]` باشد.
- **یکا (U)** باید یکی از سه مقدار زیر باشد:
  - `B`
  - `KiB`
  - `MiB`

مثال‌های معتبر:
- `37MiB`
- `512B`

مثال‌های نامعتبر:
- `12byte` (یکای مجاز نیست)
- `40000KiB` (عدد خارج از بازه‌ی مجاز)

### تبدیل واحدها
- هر `1KiB = 1024B`
- هر `1MiB = 1024KiB = 1024 × 1024B`

چون همیشه نمی‌توان حجم را دقیق نمایش داد، **باید همیشه رو به پایین گرد کنیم**.

---

## ورودی
- خط اول: عدد صحیح **T** — تعداد تست‌کیس‌ها  
```

1 ≤ T ≤ 100

```
- در هر یک از **T** خط بعد: یک عدد صحیح **m** — حجم فایل بر حسب بایت  
```

1 ≤ m ≤ 10^9

```

---

## خروجی
برای هر تست، یک خط شامل نمایش «خوانا برای انسان» فایل مطابق قوانین بالا چاپ کنید.  
پاسخ برای هر ورودی **همواره وجود دارد و یکتا است**.

---

## مثال‌ها

### ورودی نمونه ۱
```

3
29
1401
14510629

```

### خروجی نمونه ۱
```

29B
1KiB
13MiB

```

#### توضیحات نمونه
- **۲۹ بایت** → عدد در بازه‌ی مجاز → `29B`
- **۱۴۰۱ بایت** → اگر B استفاده شود عدد بزرگ‌تر از ۱۰۲۳ است → تبدیل به KiB → `1401 / 1024 = 1.x` → پایینی → `1KiB`
- **۱۴۵۱۰۶۲۹ بایت** → نیاز به MiB → `14510629 / 1048576 ≈ 13.8` → گرد به پایین → `13MiB`
