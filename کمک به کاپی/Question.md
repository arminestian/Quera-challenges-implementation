# ویروس «کاپی» و نام‌گذاری فایل‌های کپی‌شده

**محدودیت زمان:** ۰.۵ ثانیه  
**محدودیت حافظه:** ۲۵۶ مگابایت

ویروس جدیدی به جان سیستم رهنما افتاده است. نام این ویروس **«کاپی»** است و با کپی کردن فایل‌ها حافظه را پر می‌کند!  
نحوهٔ کار «کاپی» به این صورت است که ابتدا یک عدد **n** انتخاب کرده و سپس فایلی را کپی می‌کند. نام فایل جدید را با **n بار تکرار عبارت `copy of`** قبل از نام فایل اصلی می‌سازد.

به عنوان مثال، اگر فایل **you** کپی شود و مقدار n برابر **۳** باشد، خروجی چنین خواهد بود:

```

copy of copy of copy of you

```

(پس از هر عبارت `copy of` یک فاصله وجود دارد.)

با توجه به اینکه حملات آنتی‌ویروس این ویروس را ضعیف کرده است، شما باید با داشتن **نام فایل اصلی** و **تعداد تکرار n**، نام فایل نهایی را بسازید.

---

## ورودی
در تنها خط ورودی:

- عدد `n`  
- رشتهٔ `s` (نام فایل اولیه)

شرایط:
```

0 ≤ n ≤ 100
طول رشته s حداکثر 100
نام فایل فقط از حروف کوچک انگلیسی است.

```

---

## خروجی
نام فایل جدید پس از انجام n بار عبارت `copy of` را چاپ کنید.

---

## مثال

### ورودی نمونه ۱
```

3 copyof

```

### خروجی نمونه ۱
```

copy of copy of copy of copyof

```

### ورودی نمونه ۲
```

1 shoma

```

### خروجی نمونه ₂
```

copy of shoma
